02 — How each one thinks

The strategy behind each comparison count

Naive

Slides the pattern one position at a time and compares characters left to right, forgetting everything the moment it hits a mismatch. Simple, but on repetitive text it re-checks the same characters over and over.

Best case O(n)
Worst case O(n·m)
Extra memory O(1)

Knuth-Morris-Pratt

Pre-computes a "longest prefix that is also a suffix" table (LPS) for the pattern. On a mismatch, it uses that table to jump the pattern forward without ever re-comparing a character it has already matched.

Preprocessing O(m)
Search O(n + m)
Extra memory O(m)

Rabin-Karp

Hashes the pattern once, then slides a rolling hash across the text — updating it in O(1) per step instead of recomputing from scratch. Characters are only compared when two hashes collide.

Average case O(n + m)
Worst case O(n·m)
Extra memory O(1)
